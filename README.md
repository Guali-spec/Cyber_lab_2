# Network Discovery and Scanning with Nmap — Cybersecurity Lab

## Overview

This repository contains a hands-on lab on network reconnaissance using Nmap, a powerful open-source tool for network discovery and security auditing.

The lab demonstrates how attackers and security professionals gather information about a target network before launching an attack.

The complete lab report is provided as a PDF document.

---

## Objectives

The goals of this lab were to:

- Understand network scanning concepts
- Discover active hosts on a local network
- Identify open ports
- Detect running services and versions
- Perform operating system detection
- Conduct an aggressive reconnaissance scan
- Analyze potential security risks

---

## Tools Used

- Nmap (Network Mapper)
- Zenmap (Graphical User Interface for Nmap)
- Windows environment
- Local network test environment

---

## Lab Activities

### 1. Host Discovery

Identification of active devices on the local network using ping scanning.

### 2. Port Scanning

Detection of open ports on a target host to identify exposed services.

### 3. Service Detection

Identification of services and software running on open ports.

### 4. OS Detection

Fingerprinting of the target system’s operating system.

### 5. Aggressive Scan

Comprehensive scan including:

- Service and version detection
- Operating system fingerprinting
- Script scanning
- Traceroute analysis

---

## Ethical Notice

All scans were performed on a local network and authorized devices only.

Unauthorized scanning of systems without permission may be illegal.

## Key Takeaways

- Open ports represent potential attack surfaces
- Network reconnaissance is the first phase of cyber attacks
- Proper firewall configuration reduces system exposure
- Nmap is an essential tool for both attackers and defenders
